## 🔧 Decisões técnicas

1. **Modelo único `AIBlock` com enum `AIBlockType`** em vez de 3 tabelas separadas
   - **Motivo**: evita criar 3 tabelas pequenas, admin fica mais simples, padrão já usado em `HomeSection.type`
   - **Impacto**: frontend público filtra por type no próprio React (a rota pública já faz isso via `benefits`, `stages`, `education` pré-separados)

2. **URL `/inteligencia-artificial` por extenso** em vez de `/ia` ou `/ai`
   - **Motivo**: SEO (palavra-chave real), clareza para o usuário, diferencia de rotas futuras potenciais
   - **Label no nav**: "IA" (curto, cabe no grid)

3. **Ícones Lucide via map estático** em `AIPage.tsx` (não dinamic import)
   - **Motivo**: lista é curta (12 ícones), bundle fica estável, admin valida via select
   - **Alternativa descartada**: criar SVGs inline em `service-icons.tsx` (excessivo para essa página)

4. **`.shard` sempre `position: absolute`** em todos os cards novos
   - **Motivo**: respeitar o fix de layout recente documentado em `AI/HISTORY/22_04_2026-19_26_FIX_HEIGHTS_CARDS.md`
   - **Confirmado**: `.ai-benefit .shard`, `.ai-stage .shard`, `.ai-edu-card .shard`, `.ai-cta-card .shard` — todos absolute

5. **`align-items: stretch` + `height: 100%` + `margin-top: auto`**
   - Aplicado em `.ai-benefits-grid`, `.ai-stages-grid`, `.ai-edu-grid`
   - Garante altura uniforme entre cards irmãos e CTA/bullets alinhados no fundo

6. **Conteúdo original (não cópia literal da ateliware)**
   - Tom BSN: "IA que paga o custo nos 90 primeiros dias", "sem hype", "sem big bang", "transparência desconfortável"
   - Bullets concretos (MLOps, RAG, LGPD, guardrails, fine-tuning) que posicionam a BSN como parceira técnica, não vendedora

## 🐛 Problemas encontrados e soluções

- **Problema**: `npx prisma generate` falhou com `EPERM` — query_engine-windows.dll.node travada
  - **Causa**: backend dev (`tsx watch`) está rodando em outra janela mantendo a dll carregada
  - **Solução**: deixar o regenerate + migrate para o usuário aplicar quando parar o dev server. A rota `ai.ts` compila limpa via `tsc --noEmit` (Prisma exporta tipos a partir do schema, não do client regerado — mas em runtime precisa do client atualizado). Documentado na nota final do TASKS.md.

- **Problema**: `tsc --noEmit` do backend reporta erro em `seed.ts` sobre rootDir
  - **Causa**: pré-existente — `tsconfig.json` usa rootDir `src/` mas inclui `prisma/seed.ts`
  - **Solução**: não tocado. Script `build: "tsc || exit 0"` já absorve. Não é bloqueante.

- **Problema**: `tsc --noEmit` do frontend reporta ~30 erros pré-existentes (main.tsx, HeroSection, etc.)
  - **Causa**: pré-existentes
  - **Solução**: não tocado. `vite build` passa normalmente — é o comando de build real.
