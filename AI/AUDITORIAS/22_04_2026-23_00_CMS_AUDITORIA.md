# 📦 Lista consolidada de GAPS priorizada

## 🔴 ALTA prioridade (copy crítica da marca / conversão)

### G1 — Hero copy de cada página
**Onde**: Home `/`, Serviços `/servicos`, Soluções `/solucoes`, Sobre `/sobre`, Blog `/blog`, Contato `/contato`, Carreiras `/carreiras`, IA `/inteligencia-artificial`.
**Descrição**: Todos os H1 de hero, eyebrows e parágrafos de abertura estão hardcoded. Isso é conteúdo que muda conforme posicionamento, campanha ou sazonalidade.
**Modelo sugerido**:
```prisma
model PageHero {
  id          String   @id @default(uuid())
  pageKey     String   @unique  // "home", "services", "solutions", "about", "blog", "contact", "careers", "ai"
  eyebrow     String?
  eyebrowDot  Boolean  @default(true)  // pontinho verde pulsante
  titleHtml   String   // permite <em> e <br>
  lede        String?
  extraHtml   String?
  ctaLabel    String?
  ctaUrl      String?
  badgesJson  Json?    // array de { label, iconEmoji?, hasPulse? }
  isActive    Boolean  @default(true)
  updatedAt   DateTime @updatedAt
}
```

### G2 — Navegação do site (Header)
**Onde**: `Header.tsx`
**Descrição**: `NAV_ITEMS` array é hardcoded. Já existe `FooterMenu` no schema para o footer, mas o header usa um array próprio.
**Modelo sugerido**: Reutilizar `FooterMenu` (ou renomear para `SiteMenu` com campo `location: 'HEADER' | 'FOOTER_SERVICES' | 'FOOTER_COMPANY' | 'FOOTER_LEGAL'`).

### G3 — AI Cases
**Onde**: `AIPage.tsx`
**Descrição**: 4 cases hardcoded em `FALLBACK_CASES`. Não existe model de AICase.
**Modelo sugerido**:
```prisma
model AICase {
  id          String   @id @default(uuid())
  tag         String   // "AGENTE", "AUTOMAÇÃO", "RAG", "ML CLÁSSICO"
  title       String
  description String
  metric      String   // "-62% tempo de resposta"
  colorClass  String?  @default("a")
  isActive    Boolean  @default(true)
  order       Int
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt
  @@map("ai_cases")
}
```
+ rotas públicas/admin (similar ao AIBlock).

### G4 — Documentos legais (Privacy / Terms)
**Onde**: `PrivacyPage.tsx`, `TermsPage.tsx`
**Descrição**: Conteúdo jurídico precisa ser ajustável sem deploy. Inclui "VIGENTE DESDE", e-mail DPO, foro etc.
**Modelo sugerido**:
```prisma
model LegalDocument {
  id          String   @id @default(uuid())
  slug        String   @unique  // "privacy", "terms", "cookies"
  title       String
  effectiveDate DateTime
  contentMd   String   // markdown ou HTML sanitizado
  dpoEmail    String?
  version     String?  // "v1.2.0"
  isActive    Boolean  @default(true)
  updatedAt   DateTime @updatedAt
}
```

### G5 — PROJECT_TYPES da página de contato
**Onde**: `ContactPage.tsx`
**Descrição**: Array hardcoded `['Sob medida', 'Squad', 'Automação', 'Consultoria', 'Infra']` que ficou defasado em relação aos 11 serviços.
**Sugestão**: Gerar dinamicamente a partir de `Service` ativos (campo `subtitle` ou `title` curto).
